На главную

Для разработчиков

Диаграмма классов

Отображает ключевые сущности системы: Пользователь, Путешествие, Билет, Отель, Развлечение, Рекомендация и связи между ними. Показывает, какие данные хранятся (например, дата бронирования, цена, рейтинг) и как они взаимодействуют. Используется для проектирования базы данных и бизнес-логики.

Диаграмма компонентов

Показывает физические компоненты системы: веб-интерфейс, мобильное приложение, сервер API, базу данных.

Диаграмма пакетов

Группирует классы и компоненты в логические модули:Бронирование (билеты, отели), Рекомендации, Пользователи, Платежи. Помогает избежать хаоса в коде и соблюдать принцип разделения ответственности.

Диаграмма развертывания

Серверы, БД и устройства пользователей. Используется для планирования деплоя, масштабирования и распределения нагрузки.

Диаграмма композитной структуры

Внутреннее устройство сложных компонентов (например, сервиса бронирования).

Поведенческая диаграмма

Общее описание динамики работы системы. Используется для моделирования бизнес-процессов и сценариев взаимодействия.

Временная диаграмма

Длительность критичных процессов (поиск отелей, оплата).

Диаграмма последовательности

Пошаговые сценарии (бронирование: запрос → поиск → оплата).

Диаграмма объектов

Снимок данных в конкретный момент (например, бронь пользователя Х).

Диаграмма состояний

Жизненный цикл сущностей (статус брони: ожидание → оплата → подтверждение).

Диаграмма обзора взаимодействия

Связь сценариев (поиск → фильтрация → бронь).

Диаграмма коммуникации

Роли и их взаимодействие (Гость: поиск; Пользователь: бронь+рекомендации).
